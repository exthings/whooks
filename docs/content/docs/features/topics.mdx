---
title: Topics
description: Categorize and route events with Whooks Topics
---

Topics (frequently referred to as event types) form the foundation of event routing in Whooks and they belong to [Projects](/docs/api/projects). They allow you to categorize outgoing webhooks so that consumer endpoints only receive the specific events they care about.

When you configure Topics, you define:

* The catalog of events that endpoints can subscribe to.
* The optional JSON Schema validation for values of the `topic` field when you [Publish an Event](/docs/api/publish).
* The event categories displayed to consumers in your Whooks portal.

> Note: While strongly recommended for most applications, Topics are technically optional. If you do not configure any Topics, the `topic` field can be omitted when publishing events or registering endpoints.

## Endpoint Subscriptions

Every endpoint must declare which topics it wants to receive. Whooks will successfully deliver an event to an endpoint only when:

1. The endpoint is set to active.
2. The event's `topic` matches one of the endpoint's configured subscriptions.

To subscribe an endpoint to *all* events, use the wildcard `"*"`:

```shell
curl "https://whooks.exthings.org/v1/events" \
    -H "Accept: application/json" \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer {API_KEY}" \
    --data-binary @- << EOF
        {
        "uid": "fk0113ybcxknwrrjz3311w1v",
        "consumer_id": "consumer_01kg14nxkbfxgs1fv576yh871z",
        "project_id": "project_01kf4s8r2wf4crny60y8gbeen8",
        "topic": "transaction.pending",
        "data": {
            "id": "019bb3b7-57f7-7b28-9ad1-ce122c7a66c0",
            "status": "pending",
            "amount": "15"
        },
        "metadata": {
            "transaction_id": "01KESVHY8HACK7HGVYPRP90WS0"
        },
        "tags": ["019bb3b8-198d-7b7d-92af-d6dcb0ee7aad"]
        }
    EOF
```

## Event Fan-Out Architecture

When you publish a single event, Whooks handles the fan-out process automatically. It identifies every endpoint that subscribes to the event's topic and creates independent delivery attempts for each one. This ensures that a failure delivering to one endpoint does not impact the delivery to others.
