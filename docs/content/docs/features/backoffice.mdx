The Whooks backoffice provides an operational interface for monitoring and managing webhook activity. It helps teams understand what is happening across projects, endpoints, events, and deliveries.


## What the backoffice is used for

The backoffice is designed for internal teams that need visibility into webhook operations. It can be used to:

- Manage projects, topics and endpoints
- Monitor webhook delivery status
- Retry failed deliveries
- Resend a specific event to an endpoint
- Inspect failed deliveries
- Support customers during webhook incidents
- Understand delivery performance over time


## Operational visibility

Webhook systems can fail for many reasons. An endpoint may be offline, a customer system may return errors, or a network timeout may occur.
The backoffice helps teams identify these situations by showing delivery behavior and failure patterns.
